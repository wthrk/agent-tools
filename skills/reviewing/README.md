# Reviewing スキル

## 概要

セルフレビューと Codex レビューを並行実行し、両者の結果を突き合わせてコンセンサスを形成するデュアルレビュープロセス。

## 使用タイミング

- コードレビュー依頼時
- ドキュメントレビュー依頼時
- PR レビュー依頼時
- 客観的な評価が必要なコンテンツ全般

## プロセス

### ステップ 1: 並行レビュー

両方のレビューを並行実行する（単一メッセージで2つのツール呼び出し）。

**セルフレビューエージェント (Task ツール):**
- subagent_type: general-purpose
- 対象コンテンツのバグ、ロジックエラー、セキュリティ懸念、スタイル違反、改善点を特定
- 具体的な場所と理由を含めて報告

**Codex レビュー (Bash ツール):**
```bash
codex exec "Review the following code/content. Identify bugs, logic errors, security concerns, style violations, and improvements. Be specific with file paths and line numbers. Target: [target]"
```

重要: 真の並行実行のため、両方のツール（Task + Bash）を単一メッセージで呼び出すこと。

### ステップ 2: 突き合わせ

両方のレビュー結果を比較:

1. **一致点**: 両方が指摘した問題（高信頼度）
2. **セルフのみの指摘**: セルフレビューのみが発見した問題（妥当性を議論）
3. **Codex のみの指摘**: Codex のみが発見した問題（妥当性を議論）

相違点について:
- 両者の根拠と理由を評価
- コンテキストとプロジェクト規約を考慮
- コンセンサスを形成、または「係争中」として記録

### ステップ 3: 最終レポート

統一されたフォーマットで結果を提示:

```
## レビューサマリー

### 合意済み指摘事項（高信頼度）
- [場所]: [問題] - [推奨対応]

### 追加の指摘事項
- [場所]: [問題] - [出典: セルフ/Codex] - [推奨対応]

### 係争中のポイント（該当する場合）
- [場所]: [セルフの意見] vs [Codexの意見]
```

## ヒント

- 効率化のため、両方のレビューを直ちに並行開始する
- ファイルパスと行番号を具体的に記載する
- 重要度順に優先度付け: 致命的 > 重大 > 軽微
- 問題点だけでなく、良い点も記載する
- 大きな対象は論理的なチャンクに分割する
